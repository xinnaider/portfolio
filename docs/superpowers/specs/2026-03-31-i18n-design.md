# Internationalization (i18n) Design Spec

## Goal

Make the portfolio site fully bilingual (pt-BR / en) using URL prefix strategy, with Portuguese as default language.

## Routing

- **Strategy:** `prefix_except_default`
- `/` and `/about`, etc. → pt-BR (no prefix)
- `/en/` and `/en/about`, etc. → English
- Module generates `<link rel="alternate" hreflang="...">` automatically

## Module

**`@nuxtjs/i18n`** — official Nuxt i18n module.

### nuxt.config.ts additions

```ts
modules: ['@nuxt/eslint', '@nuxtjs/i18n'],

i18n: {
  locales: [
    { code: 'pt-BR', language: 'pt-BR', name: 'Português', file: 'pt-BR.json' },
    { code: 'en', language: 'en', name: 'English', file: 'en.json' },
  ],
  defaultLocale: 'pt-BR',
  strategy: 'prefix_except_default',
  lazy: true,
  langDir: 'locales/',
  detectBrowserLanguage: false,
}
```

- `detectBrowserLanguage: false` — no auto-redirect, user chooses via selector
- `lazy: true` — only load the active locale's JSON

## Locale Files

```
app/
  locales/
    pt-BR.json
    en.json
```

## Component Changes

Every component with hardcoded text gets updated:

| Component | Change |
|---|---|
| `pages/index.vue` | `useHead` title via `$t('meta.description')`, skip link via `$t('skip')` |
| `NavBar.vue` | All labels → `$t('nav.*')`, add language selector (PT \| EN) |
| `HeroSection.vue` | All text → `$t('hero.*')` |
| `AboutSection.vue` | Labels → `$t('about.*')`, highlights iterate over `$tm('about.highlights')` |
| `ExperienceSection.vue` | Labels → `$t('experience.*')`, jobs iterate over `$tm('experience.jobs')` |
| `TechStackSection.vue` | Labels → `$t('tech.*')`, category names via `$t('tech.categories.*')` |
| `EducationSection.vue` | Labels → `$t('education.*')`, entries iterate over `$tm('education.entries')` |
| `ContactSection.vue` | All text → `$t('contact.*')`, year interpolation via `$t('contact.copyright', { year })` |

### Structured Data Pattern

For arrays (highlights, jobs, education entries), components currently have data hardcoded in `<script setup>`. The approach:

1. Move data to locale JSON as arrays of objects
2. In the component, use `$tm('key')` to get the translated array
3. Iterate with `v-for`

Example:
```vue
<div v-for="(highlight, i) in $tm('about.highlights')" :key="i">
  <h3>{{ highlight.title }}</h3>
  <p>{{ highlight.description }}</p>
</div>
```

### Tech Stack Items

Tech item names (Java, Spring Boot, Docker, etc.) are universal and don't need translation. Only category labels get translated. The tech items array stays in the component or in a shared data file — not in locale JSONs.

## Language Selector

Added to `NavBar.vue` — a simple toggle:

```vue
<NuxtLink :to="switchLocalePath('pt-BR')">PT</NuxtLink>
<NuxtLink :to="switchLocalePath('en')">EN</NuxtLink>
```

Highlights the active locale. Uses `switchLocalePath()` from `@nuxtjs/i18n` which preserves the current route.

## SEO

Handled automatically by `@nuxtjs/i18n`:
- `<html lang="...">` updates per locale
- `<link rel="alternate" hreflang="pt-BR" href="/">` and `<link rel="alternate" hreflang="en" href="/en/">` generated automatically
- Meta description set via `useHead` with `$t('meta.description')`

## Files to Create

| File | Purpose |
|---|---|
| `app/locales/pt-BR.json` | All Portuguese strings |
| `app/locales/en.json` | All English strings |

## Files to Modify

| File | Purpose |
|---|---|
| `package.json` | Add `@nuxtjs/i18n` dependency |
| `nuxt.config.ts` | Add i18n module config, remove hardcoded `lang` and meta description |
| `app/pages/index.vue` | Replace hardcoded text with `$t()` |
| `app/components/NavBar.vue` | Replace text + add locale selector |
| `app/components/HeroSection.vue` | Replace all text |
| `app/components/AboutSection.vue` | Replace text, iterate highlights from locale |
| `app/components/ExperienceSection.vue` | Replace text, iterate jobs from locale |
| `app/components/TechStackSection.vue` | Replace labels, keep tech items in component |
| `app/components/EducationSection.vue` | Replace text, iterate entries from locale |
| `app/components/ContactSection.vue` | Replace text, use interpolation for year |

## Out of Scope

- More than 2 languages
- CMS-based translation management
- Automatic browser language detection/redirect
- CV/PDF generation in both languages (future enhancement)
